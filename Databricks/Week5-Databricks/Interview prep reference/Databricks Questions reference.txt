Delta Lake & Data Management

8.What is Delta Lake?

Delta Lake is a storage layer that brings ACID transactions to data lakes.

Benefits:

ACID transactions

Schema enforcement

Time travel

Data versioning

Example:

CREATE TABLE employees
USING DELTA
LOCATION '/mnt/data/employees'

9.Difference between Data Lake and Delta Lake

| Feature            | Data Lake | Delta Lake |
| ------------------ | --------- | ---------- |
| Transactions       | No        | Yes        |
| Schema enforcement | No        | Yes        |
| Time travel        | No        | Yes        |
| Data reliability   | Low       | High       |


ACID Transactions

10.What are ACID transactions?

ACID means:

Property	Meaning
Atomicity	All or nothing
Consistency	Data remains valid
Isolation	Transactions do not interfere
Durability	Data persists after commit

Delta Lake provides ACID transactions on data lakes.

11.What is Schema Evolution?


Schema evolution allows adding new columns to Delta tables without rewriting data.

Example:

df.write.format("delta") \
.option("mergeSchema","true") \
.mode("append") \
.save("/mnt/delta/employees")

12. What is Time Travel in Delta Lake?


Time Travel allows querying previous versions of data. we can restore data objects back using restore command .

Example:

SELECT * FROM employees VERSION AS OF 5;

Or:

SELECT * FROM employees TIMESTAMP AS OF '2024-01-01';

Uses:

Data recovery

Auditing

Debugging

13.Optimization

What is OPTIMIZE in Delta Lake?

OPTIMIZE improves query performance by compacting small files.

Example:

OPTIMIZE employees;

With Z-Order:

OPTIMIZE employees
ZORDER BY (employee_id);

ZORDER:

Optimizes data skipping by sorting files / filter operations / join /aggregate operations.

OPTIMIZE sales ZORDER BY (customer_id)

19. What is Unity Catalog?

Unity Catalog is a centralized data governance and access control system in Databricks used to manage data security, permissions, and metadata across the entire data platform.

It allows organizations to control who can access data, tables, files, and machine learning models from one central place.

It helps organizations manage:

Data access permissions

Data lineage

Data discovery

Data governance

Audit logging

This is very important in enterprises handling sensitive data.

Unity Catalog uses a 3-level namespace structure. Catalog → Schema → Table

sales_catalog.sales_schema.customer_table

Key Features of Unity Catalog:

1. Centralized Governance

One place to manage all data permissions.

2. Fine-Grained Access Control

Control access at different levels:

Catalog level

Schema level

Table level

Column level

Row level

Example:

GRANT SELECT ON TABLE sales.customers TO analyst;

3. Data Lineage

Shows how data flows between tables and pipelines.

Example:

Raw Data → ETL → Silver Table → Gold Table

4. Data Discovery

Users can easily find datasets using metadata.

5. Audit Logging

Tracks who accessed what data.
